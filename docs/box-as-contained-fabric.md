# Box as Contained Fabric

## Purpose

The entire Fabric of Work is contained in a Box.

The Box is the boundary that makes the framework operational, governable, portable, secure, and deliverable.

The Box is not static.

The Box has streams.

Without a Box, fabric spreads into undefined space.

With a Box, fabric becomes a controlled environment for trusted work.

## Core thesis

```text
Box = Containment Boundary
Fabric = Meta-model inside the boundary
Streams = Live movement inside the Box
Framework = Fabric of Work
Platform = Orchestrator of flow inside the Box
Governance = Loop inside the Box
Operators = Reconcilers inside the Box
Runtime = Observed work inside the Box
Evidence = Proof streams inside the Box
```

## What the Box contains

The Box contains:

- identity fabric
- data fabric
- work fabric
- policy fabric
- evidence fabric
- operator fabric
- runtime fabric
- marketplace fabric
- governance loops
- reconciliation loops
- partner delivery flows
- customer environment state

## Box streams

Streams are the live movement of state inside the Box.

A Box without streams is only a static package.

A Box with streams becomes an operating environment.

### Identity streams

Identity streams capture who or what is acting.

Examples:

- user sign-in
- agent authentication
- partner approval
- role change
- credential rotation
- permission grant
- permission revocation

### Data streams

Data streams capture movement and transformation of data.

Examples:

- ingestion
- classification
- enrichment
- lineage
- indexing
- retention
- deletion
- export

### Work streams

Work streams capture intentional state transitions.

Examples:

- partner application
- certification review
- customer deployment
- incident resolution
- marketplace publication
- workflow execution
- approval request

### Event streams

Event streams capture facts that occurred.

Examples:

- PR opened
- check failed
- charm hook fired
- deployment changed
- alert triggered
- evidence submitted
- certification expired

### Evidence streams

Evidence streams prove that work happened correctly.

Examples:

- CI status
- audit log
- runtime health
- review approval
- test result
- incident report
- customer handoff

### Policy streams

Policy streams capture rule changes and evaluations.

Examples:

- policy update
- access decision
- exception request
- compliance rule change
- risk classification
- enforcement result

### Runtime streams

Runtime streams capture observed operational state.

Examples:

- Juju status
- unit health
- relation change
- deployment status
- resource usage
- model drift
- service event

### Market streams

Market streams capture commercial and ecosystem movement.

Examples:

- partner listed
- solution published
- connector certified
- support owner changed
- listing suspended
- marketplace review completed

## Stream rule

Every important stream must have:

```text
source
identity
timestamp
scope
policy context
evidence link
retention rule
consumer
reconciliation path
```

